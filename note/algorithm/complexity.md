# 복잡도

* 자주 나오는 복잡도는 외우고 있으면 좋다
* 해당 문제가 얼마의 시간복잡도 안에 들어와야겠다 라는 감을 잡기 위해서는 문제를 많이 푸는 수 밖에 없다

### 빅오 표기 시간복잡도
- O(logN) = 단독으로는 잘 안나옴 (이분탐색 정도)  
- O(NlogN)= Sorting, 그리디, LIS 등  
- O(N<sup>2</sup>) = 5000~10000 (보통 DP)  
- O(N) = 10<sup>8</sup>   

- 그래프 문제 중 DFS는 O(v+e)
    - v : 정점의 갯수
    - e : 간선의 갯수
